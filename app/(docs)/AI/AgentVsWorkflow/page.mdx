  # WorkFlow 与 Agent

  ![](https://raw.githubusercontent.com/hugoiee/images/master/%E5%B7%A5%E4%BD%9C%E6%B5%81and%E6%99%BA%E8%83%BD%E4%BD%93.jpg)

  **工作流（Workflow）**：固定的流程，如流水线式的执行，每一步按照规则进行。像工厂流水线，严格按照设定好的程序操作。适合结构化、可预测的任务。
  <br/>
  **智能体（Agent）**：自主决策（不需要人工设计 Prompt 逻辑和工具调用），具有一定的灵活性和适应性，根据环境调整执行方式。像经验丰富的员工，可以根据情况灵活调整策略。适用于开放性问题。

  > 一个智能体 = 多个工作流的组合
  > 智能体可以拆解为多个子任务，而每个子任务可能是一个工作流。

  | 适用情况 | Workflow | Agent |
  | -------- | ----------------- | -------------- |
  | 任务是否明确 | 任务明确、结构化、可拆解为固定步骤 | 任务不确定、执行路径动态调整 |
  | 是否需要自主决策 | 只需要按照固定规则执行 | 需要自主判断、规划、决策 |
  | 可控性 | 高、可预测、易调试 | 低、更灵活但可能出错 |

  > 经验之谈： 先用工作流解决问题，只有在必要时才增加智能体的自主性。

  如果任务可以预先设计出合理的执行流程，就优先使用工作流，因为它更高效、低成本、可控。而如果任务复杂多变，需要 LLM 自己规划步骤，就可以使用智能体，但要注意成本和错误控制。

  更简单来说：遇到问题报错的是WorkFlow，遇到问题自己解决的才是智能体。

  <br/>
  ## WorkFlow应用场景

  几乎所有行业的操作流程只要由可重复步骤构成，均可视为工作流。一些经典场景如下：

  1. **公司采购订单**：审批流程为组织内的管理者、财务及采购团队提供支持。管理层需与财务团队完成预算审批流程，采购部门则协助确保第三方机构符合公司合作的法律要求与标准。
  2. **新员工入职**：当新员工加入团队时，必须完成入职流程，通常包括法律文书工作、获取 IT 资源访问权限以及参加与组织相关的培训和入职引导。
  3. **客户服务与支持**：工单系统是数字化工作流的一部分，专为支持团队设计，用于实时解答各类问题，并通过遵循常规步骤系统性地解决问题。
  4. **医疗健康管理**：每位到诊所就诊的患者都会遵循标准的分诊、诊断及管理流程。
  5. **制造业中的流水线与作业车间**：产品可以按照固定不变的线性步骤顺序（流水线车间）或根据制造内容通过不同机器以变化模式（作业车间）完成加工流程。工业流程是最早应用复杂工作流概念的领域，至今仍是制造业中的常见术语。

  ## Agent应用场景

  1. **客户体验**：人工智能代理可集成至网站和应用程序中，通过充当虚拟助手、提供心理健康支持、模拟面试及相关任务来提升客户体验。现有许多无需编码的模板可供用户直接使用，使得创建此类 AI 代理的过程更加便捷。
  2. **医疗保健**：人工智能代理可应用于现实世界中的多种医疗场景。多智能体系统尤其适用于此类环境下的问题解决，从急诊科患者的治疗规划到药物流程管理，这些系统为医疗专业人员节省了时间和精力，使其能专注于更紧急的任务。
  3. **应急响应**：在自然灾害发生时，Agent能运用深度学习算法从社交媒体上获取需要救援的用户信息。这些用户的位置可被标记出来，协助救援服务在更短时间内拯救更多人。因此，Agent不仅能在日常重复性任务中，也能在生死攸关的场景中极大造福人类生活。
  4. **金融与供应链**：Agent以被设计用于分析实时金融数据、预测未来市场趋势并优化供应链管理。自主 Agent的可定制性为我们提供了针对独特数据的个性化输出。在处理金融数据时，实施数据隐私的安全措施至关重要。
